## Создание компонента заголовка

- Создай компонент `Header` для отображения заголовка страницы, включающего информацию о текущем пользователе, в том числе значок пользователя, и опцию выхода из системы.
  - Создай навигационное меню со ссылками на страницы (только для авторизованных пользователей):
    - Страница существующих игр;
    - Страница создания игры.

## Создание компонента регистрации

- Создай компонент `Register` для страницы регистрации:
  - Должно быть поле для ввода логина, поле для ввода пароля и поле повторного ввода пароля для подтверждения.
  - Поля, связанные с паролем, должны быть с возможностью скрытия вводимых символов.
  - Создай кнопку регистрации:
    - При нажатии на кнопку регистрации должен отправляться запрос на сервер для регистрации.
    - При успешной регистрации должен произойти переход на экран авторизации.
    - При ошибке регистрации должно появляться всплывающее уведомление с причиной.
  - Создай индикатор выполнения:
    - При авторизации должен показываться индикатор выполнения.
    - При ошибке или успешной авторизации должен скрываться индикатор выполнения.
- Отображай название текущей страницы в заголовке.

## Создание компонента авторизации

- Создай компонент `Login` для страницы авторизации:
  - Должно быть поле для ввода логина и поле для ввода пароля.
  - Поле с паролем должно быть с возможностью скрытия вводимых символов.
  - Создай кнопку для регистрации:
    - При нажатии на кнопку должен осуществляться переход на экран регистрации.
  - При ошибке валидации логина и пароля должно появляться всплывающее окно.
  - Если валидация пройдена успешно, то должен отправиться запрос авторизации на сервер.
  - При ошибке авторизации должно появляться всплывающее окно с причиной.
  - Если авторизация пройдена успешно, токены и информация о пользователе должны сохраниться в базу данных, а также должен произойти переход на экран существующих игр.
- Работа с токенами:
  - Если в ответ на запрос пришел код 401, то нужно запросить новую пару токенов с помощью refreshToken и сохранить их в базу данных.
  - Если получение новых токенов прошло успешно, то должен повториться запрос.
  - Если новые токены не удалось получить или не выполнился повторный запрос, то необходимо показать всплывающее окно и перейти на экран авторизации.
  - Если время действия accessToken истекло, то должны запроситься новые токены с использованием refreshToken.
  - Если новые токены удалось получить, то их нужно сохранить в базу данных.
  - Если новые токены не получены, то необходимо показать всплывающее окно и перейти на экран авторизации.
- Операции сохранения и получения данных должны выполняться асинхронно и не блокировать главный поток.

## Создание компонента игровой доски

- Создай компонент `GameBoard` — для отображения игровой доски «Крестики-Нолики».
- Должны быть созданы отдельные модели.

## Создание компонента управления игрой

- Создай компонент `Game` — основной компонент для управления игрой, включая отображение GameBoard и логику самой игры.
- Должны быть созданы отдельные модели.
- Основная логика должна быть вынесена в сервисы (`src/services`).

## Компонент страницы существующих игр

- Создай компонент `Games` — страница существующих игр.
  - Добавь переход с экрана авторизации на экран существующих игр при успешной авторизации.
  - Создай список существующих игр:
    - Должны отображаться только игры, к которым можно присоединиться.
    - Должен отображаться UUID текущей игры.
    - Должен отображаться логин участника, который создал игру.
    - При нажатии на элемент списка на сервер должен отправиться запрос на присоединение к игре.
    - Если присоединение произошло успешно, должен быть переход на экран текущей игры.
    - При ошибке присоединения должно появляться всплывающее уведомление с причиной.
  - Создай кнопку создания игры:
    - При нажатии на кнопку должен быть переход на экран создания игры.
  - Создай кнопку выхода:
    - При нажатии на кнопку выхода должна быть полная очистка базы данных.
    - После очистки базы данных должен произойти переход на экран авторизации.
  - Добавь обновление списка текущих игр:
    - Нажатие F5 обновляет текущие данные.
    - При обновлении или при переходе на экран существующих игр должен отправиться запрос на сервер для получения списка текущих игр с использованием токена.
    - При ошибке запроса должно появляться всплывающее уведомление с причиной.

## Компонент страницы создания игры

- Создай компонент `GameCreate` — страница создания игры.
  - Создай кнопку создания игры с компьютером:
    - При нажатии на кнопку на сервер должен отправиться запрос на создании новой игры с компьютером.
    - Создай кнопку создания игры с другим игроком.
      - При нажатии на кнопку на сервер должен отправиться запрос на создании новой игры с другим игроком.
      - При успешном создании должен произойти переход на экран текущей игры.
      - При ошибке создания должно появляться всплывающее уведомление с причиной.
  - Создай индикатор выполнения:
    - При создании игры должен показываться индикатор выполнения.
    - При ошибке или успешном создании игры должен скрываться индикатор выполнения.

## Компонент страницы отображения текущей игры

- Создай компонент `ActiveGame` текущей игры:
  - Отображение UUID текущей игры.
  - Отображение логинов игроков в текущей игре.
  - Отображение значков игроков, которыми они будут ходить на игровом поле.
  - Отображение игрового поля.
    - При нажатии на определенную часть игрового поля на сервер должен отправиться запрос на изменение поля текущей игры.
    - При успешном изменении поля текущей игры должен отобразиться ход.
    - При ошибке изменения должно появляться всплывающее уведомление с причиной.
    - Каждую секунду должен отправляться на сервер запрос обновления текущей игры.
    - При успешном обновлении текущей игры информация на экране должна обновиться.
    - При ошибке обновления должно появляться всплывающее уведомление с причиной.
  - Добавь отображение текущего состояния игры:
    - Ожидание игроков — отображение текста «Ожидание игроков» с пустым игровым полем.
    - Ход игрока с UUID — если UUID совпадает с текущим авторизованным пользователем, то должно быть отображение текста «Ваш ход» и возможность хода на игровом поле, в противном случае пусть отображается текст «Ходит соперник login» и отсутствует возможность хода на игровом поле.
    - Ничья — должен отобразиться текст «Ничья» и не должно быть возможности хода на игровом поле.
    - Победа игрока с UUID — если UUID совпадает с текущим авторизованным пользователем, то должно быть отображение текста «Победа», в противном случае пусть отображается текст «Поражение» и отсутствует возможность хода на игровом поле.
- Операции отправки и сохранения данных на сервер и получения данных с сервера должны выполняться асинхронно и не блокировать главный поток.
- Можно создать другие сопутствующие необходимые компоненты.

## Настройка маршрутизации

- Настрой роутинг к страницам:
  - Регистрации;
  - Авторизации;
  - Текущей игры:
    - Маршрут должен быть доступен только для авторизованных пользователей.
  - Странице с играми:
    - Маршрут должен быть доступен только для авторизованных пользователей.
  - Создании игры:
    - Маршрут должен быть доступен только для авторизованных пользователей.

## Настройка хранилища данных

- Создай хранилище данных с помощью `vuex`.
- В хранилище должны храниться:
  - существующие игры, которые будут использоваться в качестве источника данных в компоненте, который эти данные использует;
  - другие необходимые для приложения данные.

- Добавь маршруты для страницы истории игр и для страницы лидерборда.
- Добавь в компонент заголовка и в навигационный список ссылки на страницу истории игр и страницу лидерборда.

      - Если запрос выполнился успешно, лидерборд должен сохраниться в базу данных.

- Операции отправки данных на сервер и получения данных с сервера должны выполняться асинхронно и не блокировать главный поток.
